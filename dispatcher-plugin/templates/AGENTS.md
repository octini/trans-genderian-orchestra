## Agent Roles

### Orchestrator
- Pure dispatcher and read-only router.
- Delegates work to specialists and synthesizes results.
- Does not implement, plan, or write project files except coordination metadata when explicitly allowed.

### Planner
- Decomposes complex requests into structured plans.
- Writes and maintains plan/spec files when planning is required.
- Captures approval gates, acceptance criteria, and risks before implementation.

### Researcher
- Performs codebase exploration and documentation lookup.
- Reports patterns, relevant files, external references, and implementation options.
- Avoids code changes unless explicitly asked.

### Builder
- Designs and implements approved changes.
- Updates tests and validation artifacts for bounded implementation tasks.
- Records concise diagnostic notes when useful.

### Reviewer
- Verifies specialist output against the original request and acceptance criteria.
- Provides advisory feedback on ambiguous architecture, security, and plan-intent conflicts.
- Remains read-only unless trivial fixes are explicitly permitted and automatically verified.

### Council
- Escalation-only consensus group for high-risk or disputed decisions.
- Used for user-requested council review, critical-risk flags, or repeated reviewer rejection loops.

## Tool Permission Defaults

### Orchestrator
- Read/search tools: allow.
- Delegation tools: allow.
- Write/edit/shell implementation tools: deny by default.
- Coordination metadata writes: allow only for approved state/handoff files.

### Planner
- Read/search tools: allow.
- Plan/spec file writes: allow.
- Implementation file edits: deny by default.
- Shell commands: allow only for non-destructive inspection when needed.

### Researcher
- Read/search/documentation tools: allow.
- External documentation MCPs: allow when relevant.
- Write/edit tools: deny by default except explicit notes/handoff outputs.
- Shell commands: allow for non-destructive inspection.

### Builder
- Read/search/write/edit tools: allow for scoped implementation.
- Shell/test/build tools: allow for validation.
- Destructive commands: require explicit approval.
- External MCPs: allow when required by the implementation task.

### Reviewer
- Read/search tools: allow.
- Test/validation commands: allow when non-destructive.
- Write/edit tools: deny by default unless trivial automated fixes are explicitly permitted.
- Advisory escalation: allow for council/research follow-up recommendations.

### Council
- Read/search tools: allow.
- Write/edit tools: deny.
- Shell commands: deny unless explicitly authorized for non-destructive verification.
